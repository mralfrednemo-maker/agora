Primary Pair: Intended Architecture

This is the workflow we want Agora to implement for serious deliberation.

R1 / parallel

Seed Views

Primary A, Primary B, and Secondary Seed each answer the original brief independently.

R2 / parallel

First Full Documents

Only the two primaries continue. Each primary reads all seed views and writes a full proposed document.

R3+ / alternating

Document Exchange

Primary B receives A's latest document and revises or rejects. Then Primary A receives B's latest document and does the same.

Final

Converged Text

The room stops when the two primaries accept materially the same final text, or the operator stops it.

RoundWho runs?InputRequired output
R1 Primary A, Primary B, Secondary Seed in parallel Original brief and any attached context. Independent view: answer, assumptions, risks, evidence, and important uncertainties.
R2 Primary A and Primary B in parallel Original brief, attached context, A0, B0, S0. Persistent session memory already contains prior material. Full document: agreed points, rejected/weak ideas, remaining disagreements, proposed final text, and CONVERGED or NOT_CONVERGED: reason.
R3 Primary B B's persistent context plus Primary A's latest full document. Revised full document. Accept good ideas, reject weak ideas explicitly, preserve the full agreement/disagreement register.
R4 Primary A A's persistent context plus Primary B's latest full document. Revised full document with the same required sections.
R5+ Alternates Primary B, then Primary A Only the other primary's latest full document plus a short revision instruction. Do not reattach stable context already in the session. Updated full document until both primaries converge.
Prompt rule: Every primary output after the seed round must be a self-contained full document, not a delta. It must carry forward the agreement list, disagreement list, rejected ideas, and proposed final text.

Primary Pair: Prompt Sketch

This is the prompt structure the protocol should use when implemented.

R1 seed prompt

You are {role}: Primary A / Primary B / Secondary Seed.

Read the brief and context.
Produce an independent view.

Output:
1. Working interpretation.
2. Direct answer.
3. Key assumptions.
4. Strongest supporting reasons.
5. Risks or missing context.
6. Questions that would materially change the answer.

R2 first-document prompt

You are {Primary A or Primary B}.

You have the original brief, context, and the three seed views: A0, B0, S0.
Create a full candidate final document.

Output:
1. Verdict / direct answer.
2. Agreement register.
3. Rejected or weak ideas.
4. Remaining disagreements.
5. Reasoning and evidence.
6. Proposed final text.
7. Final line: CONVERGED or NOT_CONVERGED: reason.

Alternating revision prompt

You are {Primary A or Primary B}.

Read the other primary's latest full document.
Accept improvements you agree with.
Reject ideas you disagree with and explain why.
Preserve the full document, not only changes.

Output the complete revised document with:
- agreement register
- rejected ideas
- remaining disagreements
- proposed final text
- CONVERGED or NOT_CONVERGED: reason

Convergence check

Agora should stop when both primary LLMs produce materially equivalent final text and both end with CONVERGED.

If either says NOT_CONVERGED, continue alternating until the cap or operator stop.

Code Audit Loop

This mode is already implemented as an operational relay.

Step 1 / serial

Fix

Claude Code reads the target directory and DoD, then applies changes.

Step 2 / serial

Gemini Audit

Gemini audits the target against the DoD. Any findings go back to Fix.

Step 3 / serial

Codex Validate

Codex validates after Gemini reports clean. Any findings go back to Fix.

The room finishes only after Gemini reports ZERO FINDINGS and then Codex reports ZERO FINDINGS.

If Gemini finds gaps

Agora increments the cycle and sends the room back to Claude Fix.

If Gemini is clean

Agora advances to Codex validation.

If Codex is clean

Agora marks the room complete. Auto-verdict is disabled.

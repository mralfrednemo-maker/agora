Primary Pair Convergence

Candidate replacement for the current misleading Agora Ein MDP label.

Working interpretation: three models seed the deliberation, then only the two primary models iterate on a shared final document until they converge or hit a round cap. The secondary model contributes initial perspective only.

Participants

Primary LLM A

Main author / challenger

Writes an initial view, then participates in every convergence round. It can accept, reject, or revise ideas from Primary B.

Primary LLM B

Main author / challenger

Writes an initial view, then participates in every convergence round. It receives A's documents and sends back revised documents.

Secondary LLM

Seed perspective

Writes only the first-round view. Its role is to widen the starting perspective; it does not participate in the later pair loop.

Protocol Flow

0 Optional intake

Clarify before deliberation if the ask is materially ambiguous

If the request cannot be answered without missing context, Agora should return a concise clarification request to the user before launching the room.

1 Parallel seed

All three LLMs state their independent views

A0 Primary A viewInitial answer from A
+
B0 Primary B viewInitial answer from B
+
S0 Secondary viewInitial answer from seed model

The goal is diversity of starting positions, not agreement tracking yet.

2 Primary synthesis

Only Primary A and Primary B continue

Both primary models receive all three first-round views plus their own original context. Each writes a self-contained final-draft document.

  • What I agree with across the three views
  • What I disagree with and why
  • What I reject from the other views
  • My proposed final document
3 Alternating revision loop

Pass the latest document back and forth between the two primaries

4 Final output

Stop when both primaries accept the same text or the room hits a cap

The output is the latest converged document. If convergence fails, output the latest document plus the remaining disagreement list.

Round I/O Map

R0 Intake

Input

  • User brief
  • Attachments / DoD / context
  • Selected participants and primary/secondary roles

Outcome

  • `Clarified brief` if clarification was needed
  • or `Ready brief` if no blocking ambiguity exists
R1 Parallel seed

Input

  • Ready brief
  • Attachments / source context
  • No opponent history

Outcome

  • `A0`: Primary A initial view
  • `B0`: Primary B initial view
  • `S0`: Secondary initial view
R2A A full draft

Input

  • Ready brief and attachments
  • `A0`, `B0`, `S0`
  • Primary A's own judgment

Outcome

  • `A1`: complete draft from A
  • Full agreements, rejected ideas, remaining disagreements
  • A's proposed final document
R2B B full draft

Input

  • Ready brief and attachments
  • `A0`, `B0`, `S0`
  • Primary B's own judgment

Outcome

  • `B1`: complete draft from B
  • Full agreements, rejected ideas, remaining disagreements
  • B's proposed final document
R3B B revises

Input

  • `B1`: B's own full draft
  • `A1`: A's full draft
  • Instruction: accept useful ideas, reject weak ideas

Outcome

  • `B2`: complete revised document from B
  • `CONVERGED` or `NOT_CONVERGED: reason`
R4A A revises

Input

  • `A1`: A's prior full draft
  • `B2`: B's latest revised document
  • Instruction: accept useful ideas, reject weak ideas

Outcome

  • `A2`: complete revised document from A
  • `CONVERGED` or `NOT_CONVERGED: reason`
R5B... Continue alternating

Input

  • Receiving primary's prior document
  • Other primary's latest document
  • Round cap and convergence rule

Outcome

  • `B3`, `A3`, `B4`, etc.
  • Each output is a complete document, not a delta
Ck Convergence check

Input

  • Latest primary document
  • Prior opposing primary document
  • Marker: `CONVERGED` or `NOT_CONVERGED`

Outcome

  • Stop if both primaries accept the latest substantive text
  • Otherwise continue to next alternating revision
F Final output

Input

  • Latest converged document
  • or latest non-converged document at round cap

Outcome

  • `Final document` if converged
  • `Final document + remaining disagreements` if capped

Document Contract

Every primary draft after Round 1 must be self-contained:

TITLE / SHORT VERDICT

AGREEMENTS
- Full list carried forward
- Not just new agreements from this turn

REJECTED OR WEAK IDEAS
- Idea
- Reason rejected

REMAINING DISAGREEMENTS
- Issue
- My position
- Other position

FINAL DOCUMENT
- The actual proposed answer
Each revision prompt should say:

Read the other primary model's latest document.

Accept ideas that improve the answer.
Reject ideas you disagree with and explain why.
Do not output only a critique.

Return a complete revised final document with:
- full agreements
- rejected ideas
- remaining disagreements
- final text

End with:
CONVERGED
or
NOT_CONVERGED: one-line reason

A3... Further A-side revisions until convergence or cap.
Primary B Thread B0 -> B1 -> B2 -> B3...
B0: Primary B initial view.
B1: B's full draft from A0 + B0 + S0.
B2: B revises after receiving A1. Carries full agreements, rejected ideas, remaining disagreements, and final text.
B3... Further B-side revisions until convergence or cap.
